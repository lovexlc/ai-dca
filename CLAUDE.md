# AI-DCA 项目开发指南

**项目**：美股策略助手 (AI-DCA)  
**类型**：基金投资辅助工具  
**技术栈**：React + Vite + Cloudflare Workers + D1

---

## 📋 项目概览

AI-DCA 是一个基金投资辅助工具，帮助用户进行持仓管理、基金换基策略分析、交易计划制定等。

### 核心功能模块

1. **行情中心** (51.6% 使用率) - 核心功能
   - 基金实时行情查看
   - 自选列表管理
   - 基金详情分析
   - 基金对比功能

2. **换基策略** (18.0% 使用率) - 核心功能
   - 自动化换基策略配置
   - 实时监控与执行
   - 收益回测分析

3. **转换分析** (10.0% 使用率)
   - 基金转换收益分析
   - 转换链分析
   - 转换建议

4. **持仓总览** (6.5% 使用率)
   - 持仓记录管理
   - 成本核算
   - 收益统计
   - OCR 交易记录识别

5. **交易计划** (4.6% 使用率)
   - 加仓计划
   - 定投计划
   - 卖出计划

6. **通知管理**
   - 价格提醒
   - 策略执行通知
   - 持仓异动通知

## 🚀 开发原则

### 1. 数据驱动决策

- 所有功能调整必须基于真实埋点数据
- 使用率 < 1% 的功能考虑移除或整合
- 优先优化高使用率功能（> 10%）

### 2. 做减法，不做加法

- 新功能必须有明确的用户需求支撑
- 定期清理低频功能
- 保持界面简洁，避免功能膨胀

### 3. 核心功能优先

**投入资源优先级**：
1. 行情中心（51.6%）
2. 换基策略（18.0%）
3. 转换分析（10.0%）
4. 其他功能

### 4. 小步快跑

- 快速上线 → 数据验证 → 快速调整
- 每次改动后立即监控数据变化
- 一周内完成紧急修复
- 两周内完成核心优化

## 📈 埋点规范

### 事件类型

1. **session_heartbeat**：页面停留心跳
   - `meta.activeTab`：当前 tab
   - 用于计算各页面使用率

2. **feature_event**：功能使用事件
   - `meta.feature`：功能模块
   - `meta.action`：具体操作
   - 用于计算功能使用频率

3. **action_result**：操作结果事件
   - `meta.feature`：功能模块
   - `meta.action`：具体操作
   - `meta.success`：是否成功
   - 用于计算转化率和成功率

### 关键指标

- **使用率**：基于 session_heartbeat 计算
- **转化率**：操作开始 vs 操作完成
- **成功率**：操作成功 vs 操作总数

## 🛠️ 开发流程

### 功能开发

1. **需求分析**
   - 检查埋点数据，确认用户需求
   - 评估投入产出比
   - 确定优先级

2. **设计方案**
   - 简洁优先，避免复杂设计
   - 参考高使用率功能的交互模式
   - 预留埋点位置

3. **开发实现**
   - 遵循现有代码风格
   - 添加必要的埋点
   - 编写必要的测试

4. **上线验证**
   - 灰度发布（如适用）
   - 监控埋点数据
   - 快速修复问题

5. **迭代优化**
   - 7 天后查看数据
   - 根据数据调整
   - 记录优化效果

### 功能下线

**触发条件**（满足任一）：
- 使用率 < 0.5% 持续 30 天
- 维护成本高但价值低
- 有更好的替代方案

**下线流程**：
1. 数据分析确认低频
2. 评估影响范围
3. 通知用户（如适用）
4. 代码移除
5. 监控下线后影响

### URL 设计规范

所有基金详情页面必须包含完整参数：
- 行情中心：`?tab=markets&symbol=513050`
- 持仓详情：`?tab=holdings&code=513050`
- 对比功能：`?tab=markets&symbol=513050&compare=159941,513100`

### 代码规范

- 文件行数限制：1300 行（`check_refactor_guard.mjs`）
- 超过限制需要拆分功能到独立文件
- 优先使用 URL 参数，避免 sessionStorage

### 弹窗/下拉菜单开发规范

**强制要求**：所有自定义弹窗、下拉菜单、浮层必须使用 `useClickOutside` Hook

**位置**：`src/hooks/useClickOutside.js`

**使用方法**：

```javascript
import { useClickOutside } from '../../hooks/useClickOutside.js';

function MyDropdown() {
  const [open, setOpen] = useState(false);
  const containerRef = useRef(null);
  
  // 添加点击外部关闭逻辑
  useClickOutside(containerRef, () => setOpen(false), open);
  
  return (
    <div ref={containerRef}>
      <button onClick={() => setOpen(!open)}>打开菜单</button>
      {open && (
        <div className="absolute ...">
          {/* 菜单内容 */}
        </div>
      )}
    </div>
  );
}
```

**参数说明**：
- `ref`：容器的 ref 对象
- `handler`：点击外部时的回调函数
- `enabled`：是否启用（通常传 `open` 状态）

**为什么必须使用**：
- 防止点击穿透到下层元素
- 使用 capture 阶段拦截事件
- 统一的关闭行为
- 高内聚低耦合，修改一处全局生效

**已应用的组件**：
- WatchlistSelector（自选列表下拉）
- ChartToolbarPopover（图表工具栏）
- TradePlansExperience（交易计划菜单）
- Radix Popover 组件（自动应用 `modal=true`）

**不要**：
- ❌ 不要手动监听 `mousedown` 或 `click` 事件
- ❌ 不要重复实现点击外部逻辑
- ❌ 不要忘记阻止事件传播
