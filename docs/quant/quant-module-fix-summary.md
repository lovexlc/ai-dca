| 问题 | 影响 | 严重程度 |

## 三、核心修复详解

### 3.1 回测引擎 (`src/app/backtest/index.js`)

**修复前**：
```javascript
// 错误：假设溢价差=即时收益
profit = tradeCash * netSpreadPct / 100
equity += profit  // 只加不减，永不亏损
```

**修复后**：
```javascript
// 正确：模拟真实买卖和持仓
const sellResult = simulator.executeSell(account, sellSymbol, qty, price);
const buyResult = simulator.executeBuy(account, buySymbol, qty, price);

// 权益 = 现金 + 所有持仓市值
equity = account.cash + sum(position.shares * currentPrice)
```

**关键特性**：
- ✅ `PositionState` 类追踪每个持仓
- ✅ `AccountState` 类管理现金和持仓
- ✅ `TradeSimulator` 真实模拟成交
- ✅ 计算已实现盈亏（卖出时）和未实现盈亏（持仓）
- ✅ 夏普比率、最大回撤等专业指标

### 3.2 历史数据 (`quantHistoricalData.js`)

**修复前**：
```javascript
// 纯数学函数，毫无真实性
wave = sin(index * 0.72) * 0.28 + cos(index * 0.19) * 0.12
sellPremiumPct = 0.26 + wave
```

**修复后**：
```javascript
// 基于真实市场特征的模拟
overnightMove = gaussianRandom(0, 0.015);  // 美股隔夜涨跌
morningPremium = overnightMove * 20;       // 早盘情绪溢价
intradayConverge = gaussianRandom(-0.05, 0.03); // 日内收敛
extremeEvent = 2%概率出现±1.5%极端溢价
```

**数据来源优先级**：
1. 后端API（雪球真实数据）
2. LocalStorage缓存（24小时TTL）
3. 真实特征模拟（fallback）

---

### 3.3 订单生成逻辑 (`quantOrderPlanV2.js`)

**修复前的问题**：
```javascript
// 问题1：卖出受maxOrderCash限制，导致卖少
sellQuantity = min(shares, maxOrderCash / sellPrice)

// 问题2：买入预算设上限，无法完全切换
buyBudget = min(maxOrderCash, cash + sellNet)

// 问题3：预估收益公式错误
estimatedCapture = min(sellAmount, buyAmount) * netSpreadPct / 100
```

**修复后**：
```javascript
// 修正1：卖出以目标金额为准
sellQuantity = min(shares, bidSize*0.3, targetAmount / sellPrice)

// 修正2：买入用实际卖出所得（真正的切换）
buyBudget = sellNet  // 不再受maxOrderCash限制

// 修正3：正确的收益公式
estimatedCapture = 
  (sellQty * (sellBid - sellIOPV)) -  // 卖出溢价
  (buyQty * (buyAsk - buyIOPV)) -     // 买入溢价成本
  totalFee                            // 手续费
```

**新增风控检查**：
- ✅ 极端溢价率熔断（±5%）
- ✅ IOPV比例异常检测（基差风险）
- ✅ 负溢价卖出预警
- ✅ 盘口深度不足预警

---

### 3.4 风控监控 (`quantRiskMonitor.js`)

**六大检查维度**：

1. **数据有效性**：盘口、IOPV、价格合理性
2. **市场风险**：极端溢价、基差风险、负溢价
3. **流动性风险**：盘口深度、订单占比
4. **持仓风险**：单标的集中度、总仓位
5. **交易频率**：日内次数、连续亏损
6. **损益风险**：单日亏损止损

**预警级别**：
- 🔴 **ERROR**：触发熔断，禁止交易
- 🟡 **WARNING**：风险提示，允许继续但需谨慎
- ℹ️ **INFO**：信息提示

---

### 3.5 参数优化 (`quantConfigPresets.js`)

**三种预设配置**：

| 参数 | 保守型 | 平衡型 | 激进型 |
|------|--------|--------|--------|
| 触发线 | 0.6% | 0.45% | 0.35% |
| 费用缓冲 | 0.18% | 0.15% | 0.12% |
| 单次金额 | 10000 | 16000 | 20000 |
| 冷却期 | 3天 | 2天 | 1天 |
| 最大仓位 | 50% | 70% | 90% |

**智能推荐**：
- 胜率<60% → 提高触发线
- 交易次数<5 → 降低触发线或缩短冷却期
- 回撤<-3% → 降低单次金额
- 夏普>2.0 → 可适当激进
- 费用占比>30% → 提高费用缓冲
