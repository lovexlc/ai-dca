# 切换中心

切换中心用于配置基金切换规则、查看当前命中的切换机会，并复盘已经发生的切换交易。

它不是 OCR 单次收益试算器。当前页面不上传交易截图，也没有“5 个收益卡片”和独立历史试算记录；真实切换复盘直接读取持仓成交流水。

## 页面结构

### 桌面端

页面为两列布局：

- 左侧：规则配置、Worker 自动监控、H/L 分类、实时机会和快速记录。
- 右侧：从持仓 ledger 自动推导的切换记录与收益复盘。

### 移动端

顶部只有一层子 tab：

1. **推荐机会**：只展示当前命中规则 A/B 的组合。
2. **我的方案**：管理关注组合、规则启停和全局自动监控。
3. **切换记录**：查看真实交易形成的切换链路。

## H/L 分类与持仓

- `H`：高溢价中枢基金。
- `L`：低溢价中枢基金。
- `benchmarkCodes` 是当前规则中的“我的持仓”，仅是代码字段名；UI 不使用“基准”称呼。
- `enabledCodes` 是当前规则的候选基金。
- 只有已分类且 H/L 相反的持仓与候选才形成场内候选组合。

未配置持仓时，页面会提供轻量持仓选择入口。已分类基金可设为或取消“我的持仓”，当前持仓在分类面板中有 Pin 状态标记。

## 场内规则

组合溢价差统一定义为：

```text
H 溢价率 - L 溢价率
```

两条规则的方向不同：

| 规则 | 条件 | 交易方向 |
| --- | --- | --- |
| A：差价收窄 | H-L 小于等于 `intraSellLowerPct` | 卖 L 持仓，买 H 候选 |
| B：差价扩大 | H-L 大于等于 `intraBuyOtherPct` | 卖 H 持仓，买 L 候选 |

移动端“符合规则的机会”只统计实际命中 A/B 的组合；“场内共 N 组候选”统计所有可评估的 H/L 反向组合。未达到阈值的组合不会显示“最佳机会”。

缺失的高点、历史分位、成交额等增强字段显示 `—`，不会按 `0` 展示，也不会为了补齐这些字段从列表页预取 K 线等重数据。

## 自动监控

自动监控由 notify Worker 执行：

- A 股交易时段内按分钟扫描已启用规则。
- Worker 快照是当前信号和触发方向的权威来源。
- 页面可手动运行一次，用于立即刷新快照和检查触发结果。
- “我的方案”中的监控开关是全局 Worker 开关，不承诺单个关注组合有独立后台提醒。

配置、快照与行情的主要接口：

- `/api/notify/switch/config`
- `/api/notify/switch/snapshot`
- `/api/notify/switch/run`
- `/api/markets/fund-metrics`

## 我的方案

“加入我的方案”把组合保存到浏览器 localStorage：

```text
aiDcaSwitchWatchlist
```

关注项保存卖出/买入方向、H/L 分类、规则、阈值和最近可用指标。关注本身是本地清单；后台通知是否运行由全局 Worker 监控开关和规则启停状态决定。

## 快速记录与切换记录

“快速记录”会写入一对真实交易：

- 一笔 `SELL`。
- 一笔 `BUY`。
- `SELL.switchPairId` 指向 BUY 的交易 ID。
- `BUY.switchPairId` 指向 SELL 的交易 ID。

保存后会触发同窗口 ledger 更新事件，桌面右侧复盘会立即刷新；移动端切到“切换记录”也能看到新链路。

切换记录要求卖出基金在卖出日前存在可回溯的买入交易。链路按交易日期串联，收益口径与持仓 ledger 一致：场内使用交易价/实时价格，场外使用净值。

## 数据与缓存边界

- 列表/机会页使用 Worker 快照、基金指标和已有轻量缓存。
- 不为每只基金批量请求 K 线、净值历史、财务报表或 AI 详情。
- 详情中当前没有数据源的指标显示 `—`，不触发重接口兜底。
- 快速记录和切换复盘统一写入 `aiDcaFundHoldingsLedger`，不维护第二份交易历史。
