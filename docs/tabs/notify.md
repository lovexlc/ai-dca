# 通知设置

把交易计划、切换信号和持仓收益提醒推送到常用终端。当前支持三类通道：

- iOS Bark
- Server酱³
- PC 浏览器通知 / WebSocket 实时通道

旧 Android GCM/FCM App 已废弃，页面不再提供 APK 下载、设备 ID 配对或 FCM 校验入口。

## 页面结构

顶部 3 张数字卡：

| 卡片 | 含义 |
| --- | --- |
| **通道状态** | 当前浏览器是否已经配置至少一个通知通道 |
| **Server酱³** | 当前 client 是否已保存 Server酱³ 配置 |
| **iOS Bark** | 当前 client 是否已保存 Bark Device Key |

中间 sub-tab：**iOS / Server酱³ / PC 浏览器**，各自的配置入口分开。

## 常见操作

### 配置 iOS Bark

1. 在 App Store 安装 Bark。
2. 打开 Bark，复制 `api.day.app` 开头的完整链接，或只复制 Device Key。
3. 切到通知页 iOS tab，把内容粘到输入框，保存。
4. 点「发送测试」验证。

### 配置 Server酱³

1. 打开通知页 Server酱³ tab。
2. 填入 UID 和 SendKey。
3. 保存后，测试通知会通过 Server酱³ 发送。

### 配置 PC 浏览器通知

1. 打开通知页 PC 浏览器 tab。
2. 授权浏览器通知。
3. 开启通知推送并发送本地测试通知。

## 常见问题

**Q：测试历史里我刚刚发的测试找不到了？**  
A：测试事件只保留 30 分钟，再发一次即可。

**Q：我把计划改了，怎么提醒还是按老规则推？**  
A：每次改完计划要回本页点「同步计划」，Worker 端缓存的是上次同步的版本。

**Q：我想关掉所有通知一段时间？**  
A：临时关闭系统通知权限，或在通知页清空 Bark / Server酱³ 配置并关闭 PC 通知。
